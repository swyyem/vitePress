# 核心思想：解决“回调地狱”的利器

- 想象一下现实生活中的场景：
  - **_场景：_** 你要做饭，步骤是：1. 烧水 -> 2. 煮面 -> 3. 吃面。
  - **_没有Promise（回调地狱）：_** 你开始烧水，然后必须死死盯着水壶，水一开，你立刻去煮面，然后再死死盯着锅，面一好，你立刻去吃。整个过程你什么也干不了，完全被“绑定”在了厨房。
  - **_有Promise：_** 你开始烧水，然后跟水壶说：**_“水开了叫我”_**。然后你就可以去玩手机了。水壶（Promise）在水开后**_会通知你_**，你再去煮面，然后跟锅说：“**_面好了叫我_**”。你又可以去玩手机了。

- **_Promise 就是一个“承诺”，它表示一个异步操作最终会完成（成功）或失败（拒绝），并在完成后给你一个结果。_**

## 三种状态：就像等快递

一个 Promise 对象有三种状态，**_一旦状态改变，就不可逆_**：

- `pending`**_(等待中)_** - **_比喻：_** 你的快递已经发货，正在路上。- **_初始状态，既没成功也没失败。_**
  初始状态， neither fulfilled nor rejected.
- `fulfilled`（已成功）：操作成功完成。
- `rejected`（已拒绝）：操作失败。
